# 演示视频录制与剪辑指南

> 竞赛要求：MP4 / ≤3分钟 / ≤300MB / 重点演示 / 匿名

## 拍摄脚本（3 分钟内）

| 时间段 | 时长 | 画面 | 说明 |
|:---:|:---:|------|------|
| 0:00-0:02 | 2s | 大赛统一封面 | 官网下载，插入片头 |
| 0:02-0:15 | 13s | 整机实物特写 | 正面 + 45° 斜拍，展示所有模块接线。标注"基于STM32H7的智能货物追踪系统" |
| 0:15-0:30 | 15s | NFC 激活演示 | 手机/标签靠近 PN532 → LED 变化 → 设备被唤醒 |
| 0:30-1:00 | 30s | GPS 定位演示 | 电脑屏幕串口输出 + 手机 dashboard 地图 marker 移动（双画面） |
| 1:00-1:20 | 20s | 温湿度监测 | 手指靠近 SHT31 → 串口温度上升 → dashboard 仪表盘数字变化 |
| 1:20-1:45 | 25s | 4G 数据上报 | dashboard 实时显示 telemetry JSON + 地图更新。展示数据日志滚动 |
| 1:45-2:10 | 25s | 云端指令 | 点击「暂留」→ 设备 LED 变化 / 点击「退货」→ 蜂鸣器响 / 点击「继续运输」→ 恢复正常 |
| 2:10-2:30 | 20s | 低功耗 + 离线缓存 | 快速展示：NFC 再次刷卡 → 设备进入休眠（低功耗）/ 或展示 Flash 离线存储恢复 |
| 2:30-2:50 | 20s | 全链路回顾 | 快速混剪：NFC→GPS→温度→上传→指令→休眠，展示全闭环 |
| 2:50-3:00 | 10s | 结尾 | "谢谢观看" / 不出现学校和姓名 |

## 拍摄准备

| 道具 | 用途 |
|------|------|
| 手机三脚架 / 支架 | 固定机位拍实物 |
| 电脑屏幕录制 | OBS Studio（免费）录串口 + dashboard |
| 充足光线 | 实物接线拍清楚 |
| 安静环境 | 蜂鸣器声音录进去 |

## 推荐工具

| 工具 | 用途 | 平台 |
|------|------|------|
| OBS Studio | 屏幕录制（串口+dashboard） | Win/Mac 免费 |
| 剪映 / CapCut | 视频剪辑拼接 | Win/Mac/手机 免费 |
| 手机相机 | 实物拍摄 | — |

## 剪辑流程

```
1. 手机拍实物部分（NFC激活、LED变化、蜂鸣器）
2. OBS 录屏幕部分（串口输出、dashboard 仪表盘）
3. 所有素材导入剪映
4. 按脚本时间线排列片段
5. 加速无聊等待部分（如 GPS 定位等待 1.5x~2x）
6. 加简单字幕标注关键步骤
7. 插入大赛封面（片头 2s）
8. 检查：无学校名称/logo/校徽/导师信息
9. 导出：MP4, 1080p, 30fps, H.264, ≤300MB
```

## 双画面技巧（推荐）

GPS 定位段和云端指令段用**画中画**：
- 主画面：实物设备（手机拍）
- 小窗：dashboard 屏幕（OBS 录）
- 同时看到实物动作 + 数据变化，效果最佳

剪映操作：导入两个素材 → 小窗素材拖到上层轨道 → 缩放+拖动到角落。

## 注意事项

- ⚠️ **匿名**：视频中不能出现学校名称、logo、校徽、地图上的学校位置
- ⚠️ **封面**：使用大赛提供的官方封面图片
- ⚠️ **重点在演示**：70% 以上是实物操作+屏幕展示，不要做成 PPT 讲解视频
- ⚠️ **声音**：蜂鸣器报警声是有力证明硬件在真的工作，务必录进去
- 背景干净，减少干扰
- 所有文字标注检查无学校信息
